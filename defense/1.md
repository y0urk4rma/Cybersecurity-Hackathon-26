# Process Hollow Detection Engine

A Windows C implementation of a process hollowing detection engine using
multi-signal analysis across five independent detection checks.

---

## Architecture

```
hollow_detect/
├── main.c           — CLI entry point, argument parsing
├── engine.c         — Core: init, process enumeration, run loop
├── analyzer.c       — Per-process orchestrator
├── checks.c         — All 5 detection checks
├── pe_utils.c       — PE header I/O, SHA-256 hashing
├── process_utils.c  — Image base (PEB), path, privilege utils
├── report.c         — Console output + JSON export
├── log.c            — Colour-coded, timestamped logging
├── hollow_detect.h  — Shared types, structs, prototypes
└── Makefile
```

---

## Detection Checks

| # | Check | Signal | Score |
|---|-------|--------|-------|
| 1 | SHA-256 of in-memory image ≠ on-disk file | `IMAGE_HASH_MISMATCH` | +40 |
| 2 | `MEM_PRIVATE` (not `MEM_IMAGE`) at image base | `PRIVATE_MEM_AT_IMAGE_BASE` | +35 |
| 3 | Thread start address in anonymous/private executable memory | `THREAD_START_IN_ANONYMOUS_REGION` | +30 |
| 4 | RWX page at image base | `RWX_PAGE_AT_IMAGE_BASE` | +25 |
| 4b | PE `OptionalHeader.ImageBase` ≠ actual load address | `IMAGEBASE_VALUE_MISMATCH` | +15 |
| 4c | Unreadable/corrupt PE headers in memory | `PE_HEADER_CORRUPT` | +20 |

**Verdict thresholds:**

| Score | Verdict |
|-------|---------|
| < 30 | CLEAN |
| 30–59 | SUSPICIOUS |
| 60–89 | HIGH |
| ≥ 90 | CRITICAL |

## Build

### MinGW-w64 (recommended)
```bat
make
```
or manually:
```bat
gcc -O2 -D_WIN32_WINNT=0x0601 -o hollow_detect.exe ^
    main.c engine.c analyzer.c checks.c pe_utils.c process_utils.c report.c log.c ^
    -lntdll -lpsapi -ladvapi32
```

### MSVC (Visual Studio Developer Command Prompt)
```bat
cl /O2 /D_WIN32_WINNT=0x0601 ^
   main.c engine.c analyzer.c checks.c pe_utils.c process_utils.c report.c log.c ^
   /Fe:hollow_detect.exe /link ntdll.lib psapi.lib advapi32.lib
```

> **Note:** The `-lntdll` import lib is included in MinGW-w64 sysroot.
> For MSVC, it ships with the Windows SDK.

## Usage

```
hollow_detect.exe [options]

  -p <pid>   Scan a single process by PID
  -v         Verbose / debug output
  -s         Skip system processes (PID 0, 4)
  -j <file>  Write JSON report to <file>
  -l <file>  Write log to <file>
  -a         Print ALL results (default: flagged only)
  -h         Show help
```

### Examples

```bat
REM Scan all processes (as Administrator):
hollow_detect.exe

REM Scan a single suspicious process:
hollow_detect.exe -p 1234

REM Verbose scan with JSON output:
hollow_detect.exe -v -j report.json -l scan.log

REM Show all processes including clean ones:
hollow_detect.exe -a
```

## Key Notes

### False positives to be aware of
- **ASLR** causes `IMAGEBASE_VALUE_MISMATCH` on almost every process — this
  check is scored low (15) and should only contribute as a corroborating signal.
- **Packed executables** (UPX, Themida, etc.) will trigger `IMAGE_HASH_MISMATCH`
  because their runtime-unpacked image differs from the on-disk compressed form.
  Whitelist known packed binaries if needed.
- **Self-modifying or JIT code** may also show hash mismatches.

### Privilege requirements
Run as **Administrator** to obtain `SeDebugPrivilege`, which allows opening
`PROCESS_VM_READ` on protected processes. Without it, many system processes
will return `Access Denied` and be skipped.

### Architecture note
The engine is compiled as a 64-bit binary and will inspect 64-bit processes
natively. Inspecting **32-bit (WOW64)** processes from a 64-bit binary has
limitations — `ReadProcessMemory` still works but PEB offsets differ.
The `proc_get_image_base()` function uses compile-time `#ifdef _WIN64` to
select the correct PEB offset.

## Extending the Engine

To add a new check:

1. Add a new `SIG_*` entry to the `SIGNAL_TYPE` enum in `hollow_detect.h`
2. Add its name to `SIGNAL_NAMES[]` and score constant
3. Implement `check_your_new_signal()` in `checks.c`
4. Call it from `analyze_process()` in `analyzer.c`
